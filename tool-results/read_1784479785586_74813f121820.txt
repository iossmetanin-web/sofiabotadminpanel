     1‚Üí     1‚Üí# Detailed Layer Guide for Telegram Bots
     2‚Üí     2‚Üí
     3‚Üí     3‚Üí## Overview
     4‚Üí     4‚Üí
     5‚Üí     5‚ÜíThis document provides an in-depth examination of each layer in a Clean Architecture Telegram bot project. While the main [Clean Architecture README](./README.md) established the principles and dependency rule, this guide focuses on the specific patterns, data structures, and implementation details that belong in each layer. Every code example uses Python with aiogram 3.x conventions and demonstrates how data flows across layer boundaries.

     7‚Üí     7‚Üí---
     8‚Üí     8‚Üí
     9‚Üí     9‚Üí## Domain Layer
    10‚Üí    10‚Üí
    11‚Üí    11‚ÜíThe Domain Layer is the innermost circle. It contains the purest expression of your bot's business logic ‚Äî entities that encode business rules, value objects that enforce invariants, and domain events that signal state changes. This layer has zero dependencies on any framework, database, or external service.
    12‚Üí    12‚Üí
    13‚Üí    13‚Üí### Entities
    14‚Üí    14‚Üí
    15‚Üí    15‚ÜíEntities are objects with distinct identities that persist over time. Two User entities with the same data but different IDs are different users. Entities encapsulate both state and behavior related to that state.
    16‚Üí    16‚Üí

    18‚Üí    18‚Üí# domain/entities/chat.py

    55‚Üí    55‚Üí            raise ChatAlreadyClosedError("Cannot mute a blocked chat")
    56‚Üí    56‚Üí        self.status = ChatStatus.MUTED

    61‚Üí    61‚Üí            raise ValueError("Chat is not muted")

    67‚Üí    67‚Üí            raise ChatAlreadyClosedError("Chat is already blocked")
    68‚Üí    68‚Üí        self.status = ChatStatus.BLOCKED

    95‚Üí    95‚Üí    user_telegram_id: int = 0

   121‚Üí   121‚Üí### Value Objects
   122‚Üí   122‚Üí
   123‚Üí   123‚ÜíValue objects have no identity ‚Äî they are defined entirely by their attributes. Two `TelegramId` objects with the same value are identical. Value objects are immutable and enforce validation at creation time.
   124‚Üí   124‚Üí
   125‚Üí   125‚Üí```python
   126‚Üí   126‚Üí# domain/value_objects/telegram_id.py
   127‚Üí   127‚Üífrom dataclasses import dataclass
   128‚Üí   128‚Üí
   129‚Üí   129‚Üí
   130‚Üí   130‚Üí@dataclass(frozen=True)
   131‚Üí   131‚Üíclass TelegramId:
   132‚Üí   132‚Üí    """Immutable value object representing a Telegram user/chat ID."""
   133‚Üí   133‚Üí    value: int
   134‚Üí   134‚Üí
   135‚Üí   135‚Üí    def __post_init__(self) -> None:
   136‚Üí   136‚Üí        if not isinstance(self.value, int):
   137‚Üí   137‚Üí            raise TypeError("Telegram ID must be an integer")
   138‚Üí   138‚Üí        if self.value <= 0:
   139‚Üí   139‚Üí            raise ValueError("Telegram ID must be a positive integer")
   140‚Üí   140‚Üí
   141‚Üí   141‚Üí    def __int__(self) -> int:
   142‚Üí   142‚Üí        return self.value
   143‚Üí   143‚Üí
   144‚Üí   144‚Üí    def __str__(self) -> str:
   145‚Üí   145‚Üí        return str(self.value)
   146‚Üí   146‚Üí```
   147‚Üí   147‚Üí
   148‚Üí   148‚Üí```python
   149‚Üí   149‚Üí# domain/value_objects/message_text.py
   150‚Üí   150‚Üífrom dataclasses import dataclass
   151‚Üí   151‚Üí
   152‚Üí   152‚Üí
   153‚Üí   153‚Üí@dataclass(frozen=True)
   154‚Üí   154‚Üíclass MessageText:
   155‚Üí   155‚Üí    """Immutable value object for validated message text."""
   156‚Üí   156‚Üí    value: str
   157‚Üí   157‚Üí
   158‚Üí   158‚Üí    MAX_LENGTH = 4096  # Telegram message limit
   159‚Üí   159‚Üí
   160‚Üí   160‚Üí    def __post_init__(self) -> None:
   161‚Üí   161‚Üí        if not isinstance(self.value, str):
   162‚Üí   162‚Üí            raise TypeError("Message text must be a string")
   163‚Üí   163‚Üí        if len(self.value) == 0:
   164‚Üí   164‚Üí            raise ValueError("Message text cannot be empty")
   165‚Üí   165‚Üí        if len(self.value) > self.MAX_LENGTH:
   166‚Üí   166‚Üí            raise ValueError(
   167‚Üí   167‚Üí                f"Message text exceeds Telegram limit of {self.MAX_LENGTH} characters"
   168‚Üí   168‚Üí            )
   169‚Üí   169‚Üí
   170‚Üí   170‚Üí    def truncate(self, max_length: int = 200) -> "MessageText":
   171‚Üí   171‚Üí        if len(self.value) <= max_length:
   172‚Üí   172‚Üí            return self
   173‚Üí   173‚Üí        return MessageText(self.value[: max_length - 3] + "...")
   174‚Üí   174‚Üí
   175‚Üí   175‚Üí    def __str__(self) -> str:
   176‚Üí   176‚Üí        return self.value
   177‚Üí   177‚Üí```

   224‚Üí   224‚Üí
   225‚Üí   225‚ÜíDomain events represent significant state changes that other parts of the system may need to react to. They are defined in the domain layer but published by the application layer.

   278‚Üí   278‚Üí
   279‚Üí   279‚Üí## Application Layer
   280‚Üí   280‚Üí
   281‚Üí   281‚ÜíThe Application Layer contains use cases, their input/output DTOs, and the port interfaces they depend on. This layer orchestrates the flow of data: it receives input from controllers, delegates work to entities and repositories, and returns structured output. It contains no business rules itself ‚Äî those live in entities ‚Äî but it encodes the application's workflow.
   282‚Üí   282‚Üí
   283‚Üí   283‚Üí### Use Cases
   284‚Üí   284‚Üí
   285‚Üí   285‚ÜíEach use case represents a single application operation. It declares its dependencies as abstract interfaces (ports) and operates purely through those interfaces.
   286‚Üí   286‚Üí
   287‚Üí   287‚Üí```python
   288‚Üí   288‚Üí# application/use_cases/start_conversation.py
   289‚Üí   289‚Üífrom abc import ABC, abstractmethod
   290‚Üí   290‚Üífrom dataclasses import dataclass
   291‚Üí   291‚Üífrom uuid import UUID
   292‚Üí   292‚Üí
   293‚Üí   293‚Üífrom domain.entities.conversation import Conversation
   294‚Üí   294‚Üífrom domain.events.domain_events import ConversationStarted
   295‚Üí   295‚Üí
   296‚Üí   296‚Üí
   297‚Üí   297‚Üí@dataclass
   298‚Üí   298‚Üíclass StartConversationRequest:
   299‚Üí   299‚Üí    """Input DTO for starting a conversation."""
   300‚Üí   300‚Üí    user_telegram_id: int
   301‚Üí   301‚Üí    initial_message: str | None = None
   302‚Üí   302‚Üí
   303‚Üí   303‚Üí
   304‚Üí   304‚Üí@dataclass
   305‚Üí   305‚Üíclass StartConversationResponse:
   306‚Üí   306‚Üí    """Output DTO for starting a conversation."""
   307‚Üí   307‚Üí    conversation_id: UUID
   308‚Üí   308‚Üí    is_resumed: bool
   309‚Üí   309‚Üí    remaining_messages: int
   310‚Üí   310‚Üí
   311‚Üí   311‚Üí
   312‚Üí   312‚Üíclass ConversationRepository(ABC):
   313‚Üí   313‚Üí    """Port: persistence for conversation entities."""
   314‚Üí   314‚Üí    @abstractmethod
   315‚Üí   315‚Üí    async def get_active_by_user(self, telegram_id: int) -> Conversation | None: ...
   316‚Üí   316‚Üí
   317‚Üí   317‚Üí    @abstractmethod
   318‚Üí   318‚Üí    async def save(self, conversation: Conversation) -> Conversation: ...
   319‚Üí   319‚Üí
   320‚Üí   320‚Üí
   321‚Üí   321‚Üíclass EventPublisher(ABC):
   322‚Üí   322‚Üí    """Port: publishes domain events to the event bus."""
   323‚Üí   323‚Üí    @abstractmethod
   324‚Üí   324‚Üí    async def publish(self, event: DomainEvent) -> None: ...
   325‚Üí   325‚Üí
   326‚Üí   326‚Üí
   327‚Üí   327‚Üíclass StartConversationUseCase:
   328‚Üí   328‚Üí    """Starts a new AI conversation or resumes an existing active one."""
   329‚Üí   329‚Üí
   330‚Üí   330‚Üí    def __init__(
   331‚Üí   331‚Üí        self,
   332‚Üí   332‚Üí        conversation_repo: ConversationRepository,
   333‚Üí   333‚Üí        event_publisher: EventPublisher,
   334‚Üí   334‚Üí    ) -> None:
   335‚Üí   335‚Üí        self._conversation_repo = conversation_repo
   336‚Üí   336‚Üí        self._event_publisher = event_publisher
   337‚Üí   337‚Üí
   338‚Üí   338‚Üí    async def execute(self, request: StartConversationRequest) -> StartConversationResponse:
   339‚Üí   339‚Üí        active = await self._conversation_repo.get_active_by_user(
   340‚Üí   340‚Üí            request.user_telegram_id
   341‚Üí   341‚Üí        )
   342‚Üí   342‚Üí        if active is not None:
   343‚Üí   343‚Üí            return StartConversationResponse(
   344‚Üí   344‚Üí                conversation_id=active.id,
   345‚Üí   345‚Üí                is_resumed=True,
   346‚Üí   346‚Üí                remaining_messages=active.remaining_messages(),
   347‚Üí   347‚Üí            )
   348‚Üí   348‚Üí
   349‚Üí   349‚Üí        conversation = Conversation(user_telegram_id=request.user_telegram_id)
   350‚Üí   350‚Üí        if request.initial_message:
   351‚Üí   351‚Üí            conversation.add_message()
   352‚Üí   352‚Üí
   353‚Üí   353‚Üí        saved = await self._conversation_repo.save(conversation)
   354‚Üí   354‚Üí
   355‚Üí   355‚Üí        await self._event_publisher.publish(
   356‚Üí   356‚Üí            ConversationStarted(
   357‚Üí   357‚Üí                conversation_id=saved.id,
   358‚Üí   358‚Üí                user_telegram_id=request.user_telegram_id,
   359‚Üí   359‚Üí            )
   360‚Üí   360‚Üí        )
   361‚Üí   361‚Üí
   362‚Üí   362‚Üí        return StartConversationResponse(
   363‚Üí   363‚Üí            conversation_id=saved.id,
   364‚Üí   364‚Üí            is_resumed=False,
   365‚Üí   365‚Üí            remaining_messages=saved.remaining_messages(),
   366‚Üí   366‚Üí        )
   367‚Üí   367‚Üí```

   371‚Üí   371‚Üífrom abc import ABC, abstractmethod

   404‚Üí   404‚Üíclass SendAIMessageUseCase:
   405‚Üí   405‚Üí    """Processes a user message through the AI and returns the response."""
   406‚Üí   406‚Üí

   486‚Üí   486‚Üí### Ports
   487‚Üí   487‚Üí
   488‚Üí   488‚ÜíPorts are abstract interfaces defined by the application layer. They represent the operations the application needs from the outside world. There are two kinds:
   489‚Üí   489‚Üí
   490‚Üí   490‚Üí- **Driven ports** (secondary): Interfaces the application uses to interact with external systems (databases, LLMs, event buses)
   491‚Üí   491‚Üí- **Driving ports** (primary): Interfaces through which external actors invoke the application (less common in Python, more explicit in Java/Spring)
   492‚Üí   492‚Üí
   493‚Üí   493‚Üí```python
   494‚Üí   494‚Üí# application/ports/repositories.py
   495‚Üí   495‚Üífrom abc import ABC, abstractmethod
   496‚Üí   496‚Üífrom uuid import UUID
   497‚Üí   497‚Üí
   498‚Üí   498‚Üífrom domain.entities.user import User
   499‚Üí   499‚Üífrom domain.entities.chat import Chat
   500‚Üí   500‚Üífrom domain.entities.conversation import Conversation
   501‚Üí   501‚Üí
   502‚Üí   502‚Üí
   503‚Üí   503‚Üíclass UserRepository(ABC):
   504‚Üí   504‚Üí    """Driven port for user persistence."""
   505‚Üí   505‚Üí
   506‚Üí   506‚Üí    @abstractmethod
   507‚Üí   507‚Üí    async def get_by_id(self, user_id: int) -> User | None: ...
   508‚Üí   508‚Üí
   509‚Üí   509‚Üí    @abstractmethod
   510‚Üí   510‚Üí    async def get_by_telegram_id(self, telegram_id: int) -> User | None: ...
   511‚Üí   511‚Üí
   512‚Üí   512‚Üí    @abstractmethod
   513‚Üí   513‚Üí    async def save(self, user: User) -> User: ...
   514‚Üí   514‚Üí
   515‚Üí   515‚Üí    @abstractmethod
   516‚Üí   516‚Üí    async def get_all_active(self, offset: int, limit: int) -> list[User]: ...
   517‚Üí   517‚Üí
   518‚Üí   518‚Üí    @abstractmethod
   519‚Üí   519‚Üí    async def count_active(self) -> int: ...
   520‚Üí   520‚Üí
   521‚Üí   521‚Üí
   522‚Üí   522‚Üíclass ChatRepository(ABC):
   523‚Üí   523‚Üí    """Driven port for chat persistence."""
   524‚Üí   524‚Üí
   525‚Üí   525‚Üí    @abstractmethod
   526‚Üí   526‚Üí    async def get_by_telegram_id(self, telegram_id: int) -> Chat | None: ...
   527‚Üí   527‚Üí
   528‚Üí   528‚Üí    @abstractmethod
   529‚Üí   529‚Üí    async def save(self, chat: Chat) -> Chat: ...
   530‚Üí   530‚Üí
   531‚Üí   531‚Üí
   532‚Üí   532‚Üíclass ConversationRepository(ABC):
   533‚Üí   533‚Üí    """Driven port for conversation persistence."""
   534‚Üí   534‚Üí
   535‚Üí   535‚Üí    @abstractmethod
   536‚Üí   536‚Üí    async def get_by_id(self, conversation_id: UUID) -> Conversation | None: ...
   537‚Üí   537‚Üí
   538‚Üí   538‚Üí    @abstractmethod
   539‚Üí   539‚Üí    async def get_active_by_user(self, telegram_id: int) -> Conversation | None: ...
   540‚Üí   540‚Üí
   541‚Üí   541‚Üí    @abstractmethod
   542‚Üí   542‚Üí    async def save(self, conversation: Conversation) -> Conversation: ...

   550‚Üí   550‚Üí## Infrastructure Layer
   551‚Üí   551‚Üí
   552‚Üí   552‚ÜíThe Infrastructure Layer provides concrete implementations of the ports defined by the application layer. It contains all code that interacts with external systems: databases, message queues, LLM APIs, Redis, and the Telegram Bot API itself.
   553‚Üí   553‚Üí
   554‚Üí   554‚Üí### Repository Implementations
   555‚Üí   555‚Üí
   556‚Üí   556‚Üí```python
   557‚Üí   557‚Üí# adapters/database/user_repository.py
   558‚Üí   558‚Üífrom sqlalchemy.ext.asyncio import AsyncSession
   559‚Üí   559‚Üífrom sqlalchemy import select, func
   560‚Üí   560‚Üí
   561‚Üí   561‚Üífrom application.ports.repositories import UserRepository
   562‚Üí   562‚Üífrom domain.entities.user import User, UserRole
   563‚Üí   563‚Üífrom adapters.database.models import UserModel
   564‚Üí   564‚Üí
   565‚Üí   565‚Üí
   566‚Üí   566‚Üíclass SQLAlchemyUserRepository(UserRepository):
   567‚Üí   567‚Üí    """Concrete implementation of UserRepository using SQLAlchemy."""
   568‚Üí   568‚Üí
   569‚Üí   569‚Üí    def __init__(self, session: AsyncSession) -> None:
   570‚Üí   570‚Üí        self._session = session
   571‚Üí   571‚Üí
   572‚Üí   572‚Üí    async def get_by_id(self, user_id: int) -> User | None:
   573‚Üí   573‚Üí        result = await self._session.execute(
   574‚Üí   574‚Üí            select(UserModel).where(UserModel.id == user_id)
   575‚Üí   575‚Üí        )
   576‚Üí   576‚Üí        row = result.scalar_one_or_none()
   577‚Üí   577‚Üí        return self._to_entity(row) if row else None
   578‚Üí   578‚Üí
   579‚Üí   579‚Üí    async def get_by_telegram_id(self, telegram_id: int) -> User | None:
   580‚Üí   580‚Üí        result = await self._session.execute(
   581‚Üí   581‚Üí            select(UserModel).where(UserModel.telegram_id == telegram_id)
   582‚Üí   582‚Üí        )
   583‚Üí   583‚Üí        row = result.scalar_one_or_none()
   584‚Üí   584‚Üí        return self._to_entity(row) if row else None
   585‚Üí   585‚Üí
   586‚Üí   586‚Üí    async def save(self, user: User) -> User:
   587‚Üí   587‚Üí        if user.id == 0:
   588‚Üí   588‚Üí            model = UserModel(
   589‚Üí   589‚Üí                telegram_id=user.telegram_id,
   590‚Üí   590‚Üí                username=user.username,
   591‚Üí   591‚Üí                role=user.role.value,
   592‚Üí   592‚Üí                is_active=user.is_active,
   593‚Üí   593‚Üí            )
   594‚Üí   594‚Üí            self._session.add(model)
   595‚Üí   595‚Üí            await self._session.flush()
   596‚Üí   596‚Üí            return self._to_entity(model)
   597‚Üí   597‚Üí        else:
   598‚Üí   598‚Üí            result = await self._session.execute(
   599‚Üí   599‚Üí                select(UserModel).where(UserModel.id == user.id)
   600‚Üí   600‚Üí            )
   601‚Üí   601‚Üí            model = result.scalar_one()
   602‚Üí   602‚Üí            model.username = user.username
   603‚Üí   603‚Üí            model.role = user.role.value
   604‚Üí   604‚Üí            model.is_active = user.is_active
   605‚Üí   605‚Üí            await self._session.flush()
   606‚Üí   606‚Üí            return self._to_entity(model)
   607‚Üí   607‚Üí
   608‚Üí   608‚Üí    async def get_all_active(self, offset: int, limit: int) -> list[User]:
   609‚Üí   609‚Üí        result = await self._session.execute(
   610‚Üí   610‚Üí            select(UserModel)
   611‚Üí   611‚Üí            .where(UserModel.is_active == True)
   612‚Üí   612‚Üí            .offset(offset)
   613‚Üí   613‚Üí            .limit(limit)
   614‚Üí   614‚Üí        )
   615‚Üí   615‚Üí        return [self._to_entity(row) for row in result.scalars().all()]
   616‚Üí   616‚Üí
   617‚Üí   617‚Üí    async def count_active(self) -> int:
   618‚Üí   618‚Üí        result = await self._session.execute(
   619‚Üí   619‚Üí            select(func.count()).where(UserModel.is_active == True)
   620‚Üí   620‚Üí        )
   621‚Üí   621‚Üí        return result.scalar_one()
   622‚Üí   622‚Üí
   623‚Üí   623‚Üí    @staticmethod
   624‚Üí   624‚Üí    def _to_entity(model: UserModel) -> User:
   625‚Üí   625‚Üí        return User(
   626‚Üí   626‚Üí            id=model.id,
   627‚Üí   627‚Üí            telegram_id=model.telegram_id,
   628‚Üí   628‚Üí            username=model.username,
   629‚Üí   629‚Üí            role=UserRole(model.role),
   630‚Üí   630‚Üí            is_active=model.is_active,
   631‚Üí   631‚Üí            created_at=model.created_at,
   632‚Üí   632‚Üí        )
   633‚Üí   633‚Üí```

   635‚Üí   635‚Üí### LLM Service Implementation
   636‚Üí   636‚Üí
   637‚Üí   637‚Üí```python
   638‚Üí   638‚Üí# adapters/llm/openai_service.py
   639‚Üí   639‚Üífrom uuid import UUID
   640‚Üí   640‚Üíimport openai
   641‚Üí   641‚Üí
   642‚Üí   642‚Üífrom application.use_cases.send_ai_message import LLMService
   643‚Üí   643‚Üí
   644‚Üí   644‚Üí
   645‚Üí   645‚Üíclass OpenAILLMService(LLMService):
   646‚Üí   646‚Üí    """Concrete implementation of LLMService using OpenAI API."""
   647‚Üí   647‚Üí
   648‚Üí   648‚Üí    def __init__(self, api_key: str, model: str = "gpt-4o-mini") -> None:
   649‚Üí   649‚Üí        self._client = openai.AsyncOpenAI(api_key=api_key)
   650‚Üí   650‚Üí        self._model = model
   651‚Üí   651‚Üí        self._conversation_history: dict[UUID, list[dict]] = {}
   652‚Üí   652‚Üí
   653‚Üí   653‚Üí    async def generate_response(
   654‚Üí   654‚Üí        self,
   655‚Üí   655‚Üí        conversation_id: UUID,
   656‚Üí   656‚Üí        user_message: str,
   657‚Üí   657‚Üí    ) -> str:
   658‚Üí   658‚Üí        if conversation_id not in self._conversation_history:
   659‚Üí   659‚Üí            self._conversation_history[conversation_id] = [
   660‚Üí   660‚Üí                {"role": "system", "content": "You are a helpful Telegram bot assistant."}
   661‚Üí   661‚Üí            ]
   662‚Üí   662‚Üí
   663‚Üí   663‚Üí        self._conversation_history[conversation_id].append(
   664‚Üí   664‚Üí            {"role": "user", "content": user_message}
   665‚Üí   665‚Üí        )
   666‚Üí   666‚Üí
   667‚Üí   667‚Üí        response = await self._client.chat.completions.create(
   668‚Üí   668‚Üí            model=self._model,
   669‚Üí   669‚Üí            messages=self._conversation_history[conversation_id],
   670‚Üí   670‚Üí            max_tokens=2048,
   671‚Üí   671‚Üí            temperature=0.7,
   672‚Üí   672‚Üí        )
   673‚Üí   673‚Üí
   674‚Üí   674‚Üí        assistant_message = response.choices[0].message.content
   675‚Üí   675‚Üí        self._conversation_history[conversation_id].append(
   676‚Üí   676‚Üí            {"role": "assistant", "content": assistant_message}
   677‚Üí   677‚Üí        )
   678‚Üí   678‚Üí
   679‚Üí   679‚Üí        return assistant_message
   680‚Üí   680‚Üí```

   682‚Üí   682‚Üí### Event Publisher Implementation
   683‚Üí   683‚Üí

   687‚Üí   687‚Üífrom application.use_cases.start_conversation import EventPublisher
   688‚Üí   688‚Üíimport json
   689‚Üí   689‚Üífrom redis.asyncio import Redis
   690‚Üí   690‚Üí
   691‚Üí   691‚Üí
   692‚Üí   692‚Üíclass RedisEventPublisher(EventPublisher):
   693‚Üí   693‚Üí    """Publishes domain events to Redis pub/sub channels."""
   694‚Üí   694‚Üí
   695‚Üí   695‚Üí    def __init__(self, redis: Redis) -> None:
   696‚Üí   696‚Üí        self._redis = redis
   697‚Üí   697‚Üí
   698‚Üí   698‚Üí    async def publish(self, event: DomainEvent) -> None:
   699‚Üí   699‚Üí        channel = f"events:{type(event).__name__}"
   700‚Üí   700‚Üí        payload = {
   701‚Üí   701‚Üí            "event_type": type(event).__name__,
   702‚Üí   702‚Üí            "occurred_at": event.occurred_at.isoformat(),
   703‚Üí   703‚Üí            "data": {
   704‚Üí   704‚Üí                k: str(v) for k, v in event.__dict__.items()
   705‚Üí   705‚Üí                if k != "occurred_at"
   706‚Üí   706‚Üí            },
   707‚Üí   707‚Üí        }
   708‚Üí   708‚Üí        await self._redis.publish(channel, json.dumps(payload))
   709‚Üí   709‚Üí```

   713‚Üí   713‚Üí## Presentation Layer
   714‚Üí   714‚Üí
   715‚Üí   715‚ÜíThe Presentation Layer contains the controllers (aiogram handlers), presenters (response formatters), keyboards, and filters. This layer is the bridge between the Telegram Bot API and your application's use cases.
   716‚Üí   716‚Üí
   717‚Üí   717‚Üí### Controllers (Message/Callback Handlers)
   718‚Üí   718‚Üí
   719‚Üí   719‚ÜíControllers are thin ‚Äî they parse incoming Telegram updates, call use cases, and delegate response formatting to presenters.
   720‚Üí   720‚Üí
   721‚Üí   721‚Üí```python
   722‚Üí   722‚Üí# adapters/telegram/controllers/chat_controller.py
   723‚Üí   723‚Üífrom aiogram import Router
   724‚Üí   724‚Üífrom aiogram.types import Message, CallbackQuery
   725‚Üí   725‚Üífrom aiogram.filters import Command
   726‚Üí   726‚Üí
   727‚Üí   727‚Üífrom application.use_cases.start_conversation import (
   728‚Üí   728‚Üí    StartConversationUseCase,
   729‚Üí   729‚Üí    StartConversationRequest,
   730‚Üí   730‚Üí)
   731‚Üí   731‚Üífrom application.use_cases.send_ai_message import (
   732‚Üí   732‚Üí    SendAIMessageUseCase,
   733‚Üí   733‚Üí    SendAIMessageRequest,
   734‚Üí   734‚Üí)
   735‚Üí   735‚Üífrom adapters.telegram.presenters.chat_presenter import ChatPresenter
   736‚Üí   736‚Üí
   737‚Üí   737‚Üí
   738‚Üí   738‚Üírouter = Router()
   739‚Üí   739‚Üí
   740‚Üí   740‚Üí
   741‚Üí   741‚Üí@router.message(Command("chat"))
   742‚Üí   742‚Üíasync def start_conversation(
   743‚Üí   743‚Üí    message: Message,
   744‚Üí   744‚Üí    start_conversation_use_case: StartConversationUseCase,
   745‚Üí   745‚Üí) -> None:
   746‚Üí   746‚Üí    """Controller: starts a new AI conversation."""
   747‚Üí   747‚Üí    request = StartConversationRequest(
   748‚Üí   748‚Üí        user_telegram_id=message.from_user.id,
   749‚Üí   749‚Üí    )
   750‚Üí   750‚Üí    response = await start_conversation_use_case.execute(request)
   751‚Üí   751‚Üí
   752‚Üí   752‚Üí    presenter = ChatPresenter()
   753‚Üí   753‚Üí    text = presenter.format_conversation_start(response)
   754‚Üí   754‚Üí    keyboard = presenter.build_conversation_keyboard(response)
   755‚Üí   755‚Üí
   756‚Üí   756‚Üí    await message.answer(text, reply_markup=keyboard)
   757‚Üí   757‚Üí
   758‚Üí   758‚Üí
   759‚Üí   759‚Üí@router.message()
   760‚Üí   760‚Üíasync def handle_chat_message(
   761‚Üí   761‚Üí    message: Message,
   762‚Üí   762‚Üí    send_ai_message_use_case: SendAIMessageUseCase,
   763‚Üí   763‚Üí    active_conversation_id: str | None,
   764‚Üí   764‚Üí) -> None:
   765‚Üí   765‚Üí    """Controller: processes a user message in an active AI conversation."""
   766‚Üí   766‚Üí    if not active_conversation_id:
   767‚Üí   767‚Üí        await message.answer(
   768‚Üí   768‚Üí            "No active conversation. Use /chat to start one."
   769‚Üí   769‚Üí        )
   770‚Üí   770‚Üí        return
   771‚Üí   771‚Üí
   772‚Üí   772‚Üí    request = SendAIMessageRequest(
   773‚Üí   773‚Üí        conversation_id=active_conversation_id,
   774‚Üí   774‚Üí        user_telegram_id=message.from_user.id,
   775‚Üí   775‚Üí        message_text=message.text,
   776‚Üí   776‚Üí    )
   777‚Üí   777‚Üí
   778‚Üí   778‚Üí    response = await send_ai_message_use_case.execute(request)
   779‚Üí   779‚Üí
   780‚Üí   780‚Üí    presenter = ChatPresenter()
   781‚Üí   781‚Üí    text = presenter.format_ai_response(response)
   782‚Üí   782‚Üí    await message.answer(text)
   783‚Üí   783‚Üí```

   785‚Üí   785‚Üí### Presenters (Response Formatters)
   786‚Üí   786‚Üí
   787‚Üí   787‚ÜíPresenters transform use case responses into human-readable Telegram messages. They handle markdown formatting, emoji, and keyboard layout.
   788‚Üí   788‚Üí
   789‚Üí   789‚Üí```python
   790‚Üí   790‚Üí# adapters/telegram/presenters/chat_presenter.py

   798‚Üí   798‚Üí    """Formats AI chat responses for Telegram display."""
   799‚Üí   799‚Üí
   800‚Üí   800‚Üí    def format_conversation_start(self, response: StartConversationResponse) -> str:
   801‚Üí   801‚Üí        if response.is_resumed:
   802‚Üí   802‚Üí            return (
   803‚Üí   803‚Üí                "üí¨ *Conversation Resumed*\n\n"

   806‚Üí   806‚Üí            )
   807‚Üí   807‚Üí        return (
   808‚Üí   808‚Üí            "üí¨ *New Conversation Started*\n\n"
   809‚Üí   809‚Üí            "I'm ready to chat! Ask me anything.\n"
   810‚Üí   810‚Üí            f"You have {response.remaining_messages} messages available.\n"
   811‚Üí   811‚Üí            "Use /end to close this conversation."
   812‚Üí   812‚Üí        )
   813‚Üí   813‚Üí
   814‚Üí   814‚Üí    def build_conversation_keyboard(
   815‚Üí   815‚Üí        self, response: StartConversationResponse
   816‚Üí   816‚Üí    ) -> InlineKeyboardMarkup:
   817‚Üí   817‚Üí        buttons = [
   818‚Üí   818‚Üí            [
   819‚Üí   819‚Üí                InlineKeyboardButton(

   823‚Üí   823‚Üí            ],
   824‚Üí   824‚Üí        ]
   825‚Üí   825‚Üí        if response.remaining_messages > 10:
   826‚Üí   826‚Üí            buttons.append([
   827‚Üí   827‚Üí                InlineKeyboardButton(
   828‚Üí   828‚Üí                    text="‚öôÔ∏è Settings",
   829‚Üí   829‚Üí                    callback_data="conversation_settings",
   830‚Üí   830‚Üí                )
   831‚Üí   831‚Üí            ])
   832‚Üí   832‚Üí        return InlineKeyboardMarkup(inline_keyboard=buttons)
   833‚Üí   833‚Üí
   834‚Üí   834‚Üí    def format_ai_response(self, response: SendAIMessageResponse) -> str:
   835‚Üí   835‚Üí        footer = f"\n\n_Messages remaining: {response.remaining_messages}_"
   836‚Üí   836‚Üí        if len(response.ai_response) + len(footer) > 4096:
   837‚Üí   837‚Üí            return response.ai_response[:4090 - len(footer)] + "..." + footer
   838‚Üí   838‚Üí        return response.ai_response + footer
   839‚Üí   839‚Üí```

   900‚Üí   900‚Üí## Cross-Cutting Concerns
   901‚Üí   901‚Üí
   902‚Üí   902‚ÜíCross-cutting concerns are aspects of the application that span multiple layers. In a Clean Architecture Telegram bot, these include logging, error handling, dependency injection, and configuration.
   903‚Üí   903‚Üí
   904‚Üí   904‚Üí### Logging
   905‚Üí   905‚Üí
   906‚Üí   906‚ÜíLogging must be structured and consistent. Define a logging protocol in the application layer and implement it in infrastructure.
   907‚Üí   907‚Üí
   908‚Üí   908‚Üí```python
   909‚Üí   909‚Üí# application/ports/logger.py
   910‚Üí   910‚Üífrom abc import ABC, abstractmethod
   911‚Üí   911‚Üífrom typing import Any
   912‚Üí   912‚Üí
   913‚Üí   913‚Üí
   914‚Üí   914‚Üíclass AppLogger(ABC):
   915‚Üí   915‚Üí    """Application-level logging port."""
   916‚Üí   916‚Üí
   917‚Üí   917‚Üí    @abstractmethod
   918‚Üí   918‚Üí    def info(self, message: str, **kwargs: Any) -> None: ...
   919‚Üí   919‚Üí
   920‚Üí   920‚Üí    @abstractmethod
   921‚Üí   921‚Üí    def error(self, message: str, **kwargs: Any) -> None: ...
   922‚Üí   922‚Üí
   923‚Üí   923‚Üí    @abstractmethod
   924‚Üí   924‚Üí    def warning(self, message: str, **kwargs: Any) -> None: ...
   925‚Üí   925‚Üí
   926‚Üí   926‚Üí    @abstractmethod
   927‚Üí   927‚Üí    def debug(self, message: str, **kwargs: Any) -> None: ...
   928‚Üí   928‚Üí
   929‚Üí   929‚Üí
   930‚Üí   930‚Üí# infrastructure/logging/logger.py
   931‚Üí   931‚Üíimport logging
   932‚Üí   932‚Üíimport structlog
   933‚Üí   933‚Üí
   934‚Üí   934‚Üí
   935‚Üí   935‚Üíclass StructlogLogger(AppLogger):
   936‚Üí   936‚Üí    """Concrete logger using structlog for structured JSON output."""
   937‚Üí   937‚Üí
   938‚Üí   938‚Üí    def __init__(self, service_name: str) -> None:
   939‚Üí   939‚Üí        structlog.configure(
   940‚Üí   940‚Üí            processors=[
   941‚Üí   941‚Üí                structlog.contextvars.merge_contextvars,
   942‚Üí   942‚Üí                structlog.processors.add_log_level,
   943‚Üí   943‚Üí                structlog.processors.TimeStamper(fmt="iso"),
   944‚Üí   944‚Üí                structlog.processors.JSONRenderer(),
   945‚Üí   945‚Üí            ],
   946‚Üí   946‚Üí        )
   947‚Üí   947‚Üí        self._logger = structlog.get_logger(service=service_name)
   948‚Üí   948‚Üí
   949‚Üí   949‚Üí    def info(self, message: str, **kwargs: Any) -> None:
   950‚Üí   950‚Üí        self._logger.info(message, **kwargs)
   951‚Üí   951‚Üí
   952‚Üí   952‚Üí    def error(self, message: str, **kwargs: Any) -> None:
   953‚Üí   953‚Üí        self._logger.error(message, **kwargs)
   954‚Üí   954‚Üí
   955‚Üí   955‚Üí    def warning(self, message: str, **kwargs: Any) -> None:
   956‚Üí   956‚Üí        self._logger.warning(message, **kwargs)
   957‚Üí   957‚Üí
   958‚Üí   958‚Üí    def debug(self, message: str, **kwargs: Any) -> None:
   959‚Üí   959‚Üí        self._logger.debug(message, **kwargs)
   960‚Üí   960‚Üí```

   962‚Üí   962‚Üí### Error Handling
   963‚Üí   963‚Üí
   964‚Üí   964‚ÜíDefine a consistent error handling strategy that translates domain exceptions into user-friendly Telegram messages.
   965‚Üí   965‚Üí
   966‚Üí   966‚Üí```python
   967‚Üí   967‚Üí# adapters/telegram/error_handler.py
   968‚Üí   968‚Üífrom aiogram.types import Update
   969‚Üí   969‚Üífrom aiogram import Router
   970‚Üí   970‚Üí
   971‚Üí   971‚Üífrom domain.exceptions.domain_exceptions import (
   972‚Üí   972‚Üí    ChatAlreadyClosedError,
   973‚Üí   973‚Üí    ConversationAlreadyEndedError,
   974‚Üí   974‚Üí    MessageLimitExceededError,
   975‚Üí   975‚Üí)
   976‚Üí   976‚Üí
   977‚Üí   977‚Üí
   978‚Üí   978‚Üírouter = Router()
   979‚Üí   979‚Üí
   980‚Üí   980‚Üí
   981‚Üí   981‚Üí@router.error()
   982‚Üí   982‚Üíasync def handle_errors(update: Update, exception: Exception) -> None:
   983‚Üí   983‚Üí    """Global error handler for the bot."""
   984‚Üí   984‚Üí
   985‚Üí   985‚Üí    error_messages = {
   986‚Üí   986‚Üí        ChatAlreadyClosedError: "This chat is no longer active.",
   987‚Üí   987‚Üí        ConversationAlreadyEndedError: "This conversation has ended. Start a new one with /chat.",
   988‚Üí   988‚Üí        MessageLimitExceededError: "You've reached the message limit for this conversation. Start a new one with /chat.",
   989‚Üí   989‚Üí        PermissionError: "You don't have permission to perform this action.",
   990‚Üí   990‚Üí        ValueError: "Invalid input. Please check your request and try again.",
   991‚Üí   991‚Üí    }
   992‚Üí   992‚Üí
   993‚Üí   993‚Üí    message_text = error_messages.get(type(exception), "An unexpected error occurred. Please try again later.")
   994‚Üí   994‚Üí
   995‚Üí   995‚Üí    if update.message:
   996‚Üí   996‚Üí        await update.message.answer(f"‚öÝÔ∏è {message_text}")
   997‚Üí   997‚Üí    elif update.callback_query:
   998‚Üí   998‚Üí        await update.callback_query.answer(f"‚öÝÔ∏è {message_text}", show_alert=True)
   999‚Üí   999‚Üí```

  1120‚Üí  1120‚Üí## Data Flow Across Layers
  1121‚Üí  1121‚Üí
  1122‚Üí  1122‚ÜíUnderstanding how data transforms as it crosses layer boundaries is critical. Here is the complete flow for a user sending a message to the AI:
  1123‚Üí  1123‚Üí
  1124‚Üí  1124‚Üí```
  1125‚Üí  1125‚Üí1. Telegram API ‚Üí aiogram receives Update
  1126‚Üí  1126‚Üí2. Framework Layer (aiogram Router) dispatches to handler
  1127‚Üí  1127‚Üí3. Presentation Layer:
  1128‚Üí  1128‚Üí   - Controller parses Message ‚Üí creates SendAIMessageRequest DTO
  1129‚Üí  1129‚Üí   - Calls SendAIMessageUseCase.execute(request)
  1130‚Üí  1130‚Üí4. Application Layer:
  1131‚Üí  1131‚Üí   - Use case loads Conversation via ConversationRepository port
  1132‚Üí  1132‚Üí   - Validates message via MessageText value object
  1133‚Üí  1133‚Üí   - Calls LLMService.generate_response() port
  1134‚Üí  1134‚Üí   - Saves updated Conversation via repository port
  1135‚Üí  1135‚Üí   - Publishes MessageSentToLLM event via EventPublisher port
  1136‚Üí  1136‚Üí   - Returns SendAIMessageResponse DTO
  1137‚Üí  1137‚Üí5. Presentation Layer:
  1138‚Üí  1138‚Üí   - Presenter formats response DTO into Telegram markdown
  1139‚Üí  1139‚Üí   - Handler calls message.answer(formatted_text)
  1140‚Üí  1140‚Üí6. Framework Layer:
  1141‚Üí  1141‚Üí   - aiogram sends API request to Telegram servers
  1142‚Üí  1142‚Üí   - User receives the AI response
  1143‚Üí  1143‚Üí```
  1144‚Üí  1144‚Üí
  1145‚Üí  1145‚ÜíAt each boundary crossing, data is transformed from one representation to another:
  1146‚Üí  1146‚Üí- **Telegram Update ‚Üí Request DTO**: Controller extracts relevant fields
  1147‚Üí  1147‚Üí- **Request DTO ‚Üí Domain Entities**: Use case reconstructs domain objects from repository
  1148‚Üí  1148‚Üí- **Domain Entities ‚Üí Response DTO**: Use case maps entity state to output structure
  1149‚Üí  1149‚Üí- **Response DTO ‚Üí Telegram Message**: Presenter formats for human consumption
  1150‚Üí  1150‚Üí
  1151‚Üí  1151‚ÜíThis transformation at each boundary is what makes Clean Architecture powerful ‚Äî each layer communicates in its own language, optimized for its own concerns.

  1155‚Üí  1155‚Üí## Common Pitfalls by Layer
  1156‚Üí  1156‚Üí
  1157‚Üí  1157‚Üí### Domain Layer Pitfalls
  1158‚Üí  1158‚Üí
  1159‚Üí  1159‚Üí- **Anemic entities**: Entities with only getters and setters, no business logic. Push validation and state transitions into entities.
  1160‚Üí  1160‚Üí- **Framework leakage**: Importing SQLAlchemy, Pydantic, or aiogram types in entity classes. Entities must be pure Python.
  1161‚Üí  1161‚Üí- **Missing value objects**: Using raw `int` for telegram IDs or raw `str` for messages instead of dedicated value objects with validation.
  1162‚Üí  1162‚Üí
  1163‚Üí  1163‚Üí### Application Layer Pitfalls
  1164‚Üí  1164‚Üí
  1165‚Üí  1165‚Üí- **Use cases with infrastructure logic**: Directly importing database or LLM clients instead of using port interfaces.
  1166‚Üí  1166‚Üí- **Fat DTOs**: Putting validation or business logic in DTOs. DTOs should be dumb data carriers.
  1167‚Üí  1167‚Üí- **Missing error handling**: Not defining what happens when a port raises an exception. Use cases should handle expected failures gracefully.
  1168‚Üí  1168‚Üí
  1169‚Üí  1169‚Üí### Infrastructure Layer Pitfalls
  1170‚Üí  1170‚Üí
  1171‚Üí  1171‚Üí- **Leaking ORM models**: Returning SQLAlchemy model objects instead of domain entities from repositories. Always convert in the adapter.
  1172‚Üí  1172‚Üí- **Tight coupling to specific vendor**: Hard-coding OpenAI-specific parameters in the LLM service. Keep vendor specifics behind the adapter.
  1173‚Üí  1173‚Üí
  1174‚Üí  1174‚Üí### Presentation Layer Pitfalls
  1175‚Üí  1175‚Üí
  1176‚Üí  1176‚Üí- **Business logic in handlers**: Handlers should parse input and format output only. Any decision-making belongs in use cases.
  1177‚Üí  1177‚Üí- **Skipping the presenter**: Formatting messages directly in handlers couples presentation to control flow.
  1178‚Üí  1178‚Üí- **Over-complex keyboard builders**: Keyboard logic should be simple and delegatable to a dedicated module.
  1179‚Üí  1179‚Üí
