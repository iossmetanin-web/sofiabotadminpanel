     1→# Detailed Layer Guide for Telegram Bots
     2→
     3→## Overview
     4→
     5→This document provides an in-depth examination of each layer in a Clean Architecture Telegram bot project. While the main [Clean Architecture README](./README.md) established the principles and dependency rule, this guide focuses on the specific patterns, data structures, and implementation details that belong in each layer. Every code example uses Python with aiogram 3.x conventions and demonstrates how data flows across layer boundaries.
     6→
     7→---
     8→
     9→## Domain Layer
    10→
    11→The Domain Layer is the innermost circle. It contains the purest expression of your bot's business logic — entities that encode business rules, value objects that enforce invariants, and domain events that signal state changes. This layer has zero dependencies on any framework, database, or external service.
    12→
    13→### Entities
    14→
    15→Entities are objects with distinct identities that persist over time. Two User entities with the same data but different IDs are different users. Entities encapsulate both state and behavior related to that state.
    16→
    17→```python
    18→# domain/entities/chat.py
    19→from dataclasses import dataclass, field
    20→from datetime import datetime
    21→from enum import Enum
    22→from uuid import UUID, uuid4
    23→
    24→from domain.value_objects.telegram_id import TelegramId
    25→from domain.value_objects.language import Language
    26→from domain.exceptions.domain_exceptions import ChatAlreadyClosedError
    27→
    28→
    29→class ChatType(Enum):
    30→    PRIVATE = "private"
    31→    GROUP = "group"
    32→    SUPERGROUP = "supergroup"
    33→    CHANNEL = "channel"
    34→
    35→
    36→class ChatStatus(Enum):
    37→    ACTIVE = "active"
    38→    MUTED = "muted"
    39→    BLOCKED = "blocked"
    40→
    41→
    42→@dataclass
    43→class Chat:
    44→    """Chat entity representing a Telegram chat with its business rules."""
    45→    id: UUID = field(default_factory=uuid4)
    46→    telegram_id: TelegramId = field(default=None)
    47→    chat_type: ChatType = ChatType.PRIVATE
    48→    status: ChatStatus = ChatStatus.ACTIVE
    49→    language: Language = field(default_factory=lambda: Language.EN)
    50→    created_at: datetime = field(default_factory=datetime.utcnow)
    51→    updated_at: datetime = field(default_factory=datetime.utcnow)
    52→
    53→    def mute(self) -> None:
    54→        if self.status == ChatStatus.BLOCKED:
    55→            raise ChatAlreadyClosedError("Cannot mute a blocked chat")
    56→        self.status = ChatStatus.MUTED
    57→        self.updated_at = datetime.utcnow()
    58→
    59→    def unmute(self) -> None:
    60→        if self.status != ChatStatus.MUTED:
    61→            raise ValueError("Chat is not muted")
    62→        self.status = ChatStatus.ACTIVE
    63→        self.updated_at = datetime.utcnow()
    64→
    65→    def block(self) -> None:
    66→        if self.status == ChatStatus.BLOCKED:
    67→            raise ChatAlreadyClosedError("Chat is already blocked")
    68→        self.status = ChatStatus.BLOCKED
    69→        self.updated_at = datetime.utcnow()
    70→
    71→    def is_group_chat(self) -> bool:
    72→        return self.chat_type in (ChatType.GROUP, ChatType.SUPERGROUP)
    73→
    74→    def set_language(self, language: Language) -> None:
    75→        self.language = language
    76→        self.updated_at = datetime.utcnow()
    77→```
    78→
    79→```python
    80→# domain/entities/conversation.py
    81→from dataclasses import dataclass, field
    82→from datetime import datetime
    83→from uuid import UUID, uuid4
    84→
    85→from domain.exceptions.domain_exceptions import (
    86→    ConversationAlreadyEndedError,
    87→    MessageLimitExceededError,
    88→)
    89→
    90→
    91→@dataclass
    92→class Conversation:
    93→    """Aggregate-like entity for an AI conversation session."""
    94→    id: UUID = field(default_factory=uuid4)
    95→    user_telegram_id: int = 0
    96→    is_active: bool = True
    97→    message_count: int = 0
    98→    max_messages: int = 100
    99→    created_at: datetime = field(default_factory=datetime.utcnow)
   100→    ended_at: datetime | None = None
   101→
   102→    def add_message(self) -> None:
   103→        if not self.is_active:
   104→            raise ConversationAlreadyEndedError("Cannot add messages to ended conversation")
   105→        if self.message_count >= self.max_messages:
   106→            raise MessageLimitExceededError(
   107→                f"Message limit of {self.max_messages} exceeded"
   108→            )
   109→        self.message_count += 1
   110→
   111→    def end(self) -> None:
   112→        if not self.is_active:
   113→            raise ConversationAlreadyEndedError("Conversation is already ended")
   114→        self.is_active = False
   115→        self.ended_at = datetime.utcnow()
   116→
   117→    def remaining_messages(self) -> int:
   118→        return max(0, self.max_messages - self.message_count)
   119→```
   120→
   121→### Value Objects
   122→
   123→Value objects have no identity — they are defined entirely by their attributes. Two `TelegramId` objects with the same value are identical. Value objects are immutable and enforce validation at creation time.
   124→
   125→```python
   126→# domain/value_objects/telegram_id.py
   127→from dataclasses import dataclass
   128→
   129→
   130→@dataclass(frozen=True)
   131→class TelegramId:
   132→    """Immutable value object representing a Telegram user/chat ID."""
   133→    value: int
   134→
   135→    def __post_init__(self) -> None:
   136→        if not isinstance(self.value, int):
   137→            raise TypeError("Telegram ID must be an integer")
   138→        if self.value <= 0:
   139→            raise ValueError("Telegram ID must be a positive integer")
   140→
   141→    def __int__(self) -> int:
   142→        return self.value
   143→
   144→    def __str__(self) -> str:
   145→        return str(self.value)
   146→```
   147→
   148→```python
   149→# domain/value_objects/message_text.py
   150→from dataclasses import dataclass
   151→
   152→
   153→@dataclass(frozen=True)
   154→class MessageText:
   155→    """Immutable value object for validated message text."""
   156→    value: str
   157→
   158→    MAX_LENGTH = 4096  # Telegram message limit
   159→
   160→    def __post_init__(self) -> None:
   161→        if not isinstance(self.value, str):
   162→            raise TypeError("Message text must be a string")
   163→        if len(self.value) == 0:
   164→            raise ValueError("Message text cannot be empty")
   165→        if len(self.value) > self.MAX_LENGTH:
   166→            raise ValueError(
   167→                f"Message text exceeds Telegram limit of {self.MAX_LENGTH} characters"
   168→            )
   169→
   170→    def truncate(self, max_length: int = 200) -> "MessageText":
   171→        if len(self.value) <= max_length:
   172→            return self
   173→        return MessageText(self.value[: max_length - 3] + "...")
   174→
   175→    def __str__(self) -> str:
   176→        return self.value
   177→```
   178→
   179→```python
   180→# domain/value_objects/language.py
   181→from dataclasses import dataclass
   182→from enum import Enum
   183→
   184→
   185→class LanguageCode(Enum):
   186→    EN = "en"
   187→    RU = "ru"
   188→    DE = "de"
   189→    FR = "fr"
   190→    ES = "es"
   191→    ZH = "zh"
   192→    JA = "ja"
   193→    KO = "ko"
   194→
   195→
   196→@dataclass(frozen=True)
   197→class Language:
   198→    """Immutable value object representing a language preference."""
   199→    code: LanguageCode = LanguageCode.EN
   200→
   201→    @classmethod
   202→    def from_iso(cls, iso_code: str) -> "Language":
   203→        try:
   204→            return cls(code=LanguageCode(iso_code.lower()))
   205→        except ValueError:
   206→            raise ValueError(f"Unsupported language code: {iso_code}")
   207→
   208→    @property
   209→    def display_name(self) -> str:
   210→        names = {
   211→            LanguageCode.EN: "English",
   212→            LanguageCode.RU: "Русский",
   213→            LanguageCode.DE: "Deutsch",
   214→            LanguageCode.FR: "Français",
   215→            LanguageCode.ES: "Español",
   216→            LanguageCode.ZH: "中文",
   217→            LanguageCode.JA: "日本語",
   218→            LanguageCode.KO: "한국어",
   219→        }
   220→        return names[self.code]
   221→```

   223→### Domain Events
   224→
   225→Domain events represent significant state changes that other parts of the system may need to react to. They are defined in the domain layer but published by the application layer.
   226→
   227→```python
   228→# domain/events/domain_events.py
   229→from dataclasses import dataclass, field
   230→from datetime import datetime
   231→from uuid import UUID
   232→
   233→
   234→@dataclass
   235→class DomainEvent:
   236→    """Base class for all domain events."""
   237→    occurred_at: datetime = field(default_factory=datetime.utcnow)
   238→
   239→
   240→@dataclass
   241→class UserRegistered(DomainEvent):
   242→    """Fired when a new user registers via the bot."""
   243→    user_id: UUID = field(default=None)
   244→    telegram_id: int = 0
   245→    username: str = ""
   246→
   247→
   248→@dataclass
   249→class ConversationStarted(DomainEvent):
   250→    """Fired when a user starts a new AI conversation."""
   251→    conversation_id: UUID = field(default=None)
   252→    user_telegram_id: int = 0
   253→
   254→
   255→@dataclass
   256→class ConversationEnded(DomainEvent):
   257→    """Fired when a conversation ends."""
   258→    conversation_id: UUID = field(default=None)
   259→    user_telegram_id: int = 0
   260→    message_count: int = 0
   261→
   262→
   263→@dataclass
   264→class MessageSentToLLM(DomainEvent):
   265→    """Fired when a message is sent to the LLM for processing."""
   266→    conversation_id: UUID = field(default=None)
   267→    user_telegram_id: int = 0
   268→    token_count: int = 0
   269→
   270→
   271→@dataclass
   272→class UserBlockedBot(DomainEvent):
   273→    """Fired when a user blocks the bot (detected via failed message delivery)."""
   274→    telegram_id: int = 0
   275→```

   278→
   279→## Application Layer
   280→
   281→The Application Layer contains use cases, their input/output DTOs, and the port interfaces they depend on. This layer orchestrates the flow of data: it receives input from controllers, delegates work to entities and repositories, and returns structured output. It contains no business rules itself — those live in entities — but it encodes the application's workflow.
   282→
   283→### Use Cases
   284→
   285→Each use case represents a single application operation. It declares its dependencies as abstract interfaces (ports) and operates purely through those interfaces.
   286→
   287→```python
   288→# application/use_cases/start_conversation.py
   289→from abc import ABC, abstractmethod
   290→from dataclasses import dataclass
   291→from uuid import UUID
   292→
   293→from domain.entities.conversation import Conversation
   294→from domain.events.domain_events import ConversationStarted
   295→
   296→
   297→@dataclass
   298→class StartConversationRequest:
   299→    """Input DTO for starting a conversation."""
   300→    user_telegram_id: int
   301→    initial_message: str | None = None
   302→
   303→
   304→@dataclass
   305→class StartConversationResponse:
   306→    """Output DTO for starting a conversation."""
   307→    conversation_id: UUID
   308→    is_resumed: bool
   309→    remaining_messages: int
   310→
   311→
   312→class ConversationRepository(ABC):
   313→    """Port: persistence for conversation entities."""
   314→    @abstractmethod
   315→    async def get_active_by_user(self, telegram_id: int) -> Conversation | None: ...
   316→
   317→    @abstractmethod
   318→    async def save(self, conversation: Conversation) -> Conversation: ...
   319→
   320→
   321→class EventPublisher(ABC):
   322→    """Port: publishes domain events to the event bus."""
   323→    @abstractmethod
   324→    async def publish(self, event: DomainEvent) -> None: ...
   325→
   326→
   327→class StartConversationUseCase:
   328→    """Starts a new AI conversation or resumes an existing active one."""
   329→
   330→    def __init__(
   331→        self,
   332→        conversation_repo: ConversationRepository,
   333→        event_publisher: EventPublisher,
   334→    ) -> None:
   335→        self._conversation_repo = conversation_repo
   336→        self._event_publisher = event_publisher
   337→
   338→    async def execute(self, request: StartConversationRequest) -> StartConversationResponse:
   339→        active = await self._conversation_repo.get_active_by_user(
   340→            request.user_telegram_id
   341→        )
   342→        if active is not None:
   343→            return StartConversationResponse(
   344→                conversation_id=active.id,
   345→                is_resumed=True,
   346→                remaining_messages=active.remaining_messages(),
   347→            )
   348→
   349→        conversation = Conversation(user_telegram_id=request.user_telegram_id)
   350→        if request.initial_message:
   351→            conversation.add_message()
   352→
   353→        saved = await self._conversation_repo.save(conversation)
   354→
   355→        await self._event_publisher.publish(
   356→            ConversationStarted(
   357→                conversation_id=saved.id,
   358→                user_telegram_id=request.user_telegram_id,
   359→            )
   360→        )
   361→
   362→        return StartConversationResponse(
   363→            conversation_id=saved.id,
   364→            is_resumed=False,
   365→            remaining_messages=saved.remaining_messages(),
   366→        )
   367→```
   368→
   369→```python
   370→# application/use_cases/send_ai_message.py
   371→from abc import ABC, abstractmethod
   372→from dataclasses import dataclass
   373→from uuid import UUID
   374→
   375→from domain.entities.conversation import Conversation
   376→from domain.value_objects.message_text import MessageText
   377→from domain.events.domain_events import MessageSentToLLM
   378→
   379→
   380→@dataclass
   381→class SendAIMessageRequest:
   382→    conversation_id: UUID
   383→    user_telegram_id: int
   384→    message_text: str
   385→
   386→
   387→@dataclass
   388→class SendAIMessageResponse:
   389→    ai_response: str
   390→    conversation_id: UUID
   391→    remaining_messages: int
   392→
   393→
   394→class LLMService(ABC):
   395→    """Port: abstract interface for language model interaction."""
   396→    @abstractmethod
   397→    async def generate_response(
   398→        self,
   399→        conversation_id: UUID,
   400→        user_message: str,
   401→    ) -> str: ...
   402→
   403→
   404→class SendAIMessageUseCase:
   405→    """Processes a user message through the AI and returns the response."""
   406→
   407→    def __init__(
   408→        self,
   409→        conversation_repo: ConversationRepository,
   410→        llm_service: LLMService,
   411→        event_publisher: EventPublisher,
   412→    ) -> None:
   413→        self._conversation_repo = conversation_repo
   414→        self._llm_service = llm_service
   415→        self._event_publisher = event_publisher
   416→
   417→    async def execute(self, request: SendAIMessageRequest) -> SendAIMessageResponse:
   418→        conversation = await self._conversation_repo.get_by_id(request.conversation_id)
   419→        if conversation is None:
   420→            raise ValueError(f"Conversation {request.conversation_id} not found")
   421→
   422→        message = MessageText(request.message_text)
   423→        conversation.add_message()
   424→
   425→        ai_response = await self._llm_service.generate_response(
   426→            conversation_id=conversation.id,
   427→            user_message=message.value,
   428→        )
   429→
   430→        await self._conversation_repo.save(conversation)
   431→
   432→        await self._event_publisher.publish(
   433→            MessageSentToLLM(
   434→                conversation_id=conversation.id,
   435→                user_telegram_id=request.user_telegram_id,
   436→                token_count=len(message.value.split()),  # rough estimate
   437→            )
   438→        )
   439→
   440→        return SendAIMessageResponse(
   441→            ai_response=ai_response,
   442→            conversation_id=conversation.id,
   443→            remaining_messages=conversation.remaining_messages(),
   444→        )
   445→```

   447→### DTOs (Data Transfer Objects)
   448→
   449→DTOs are flat data structures that carry data across layer boundaries. They contain no business logic — only fields and optionally validation metadata.
   450→
   451→```python
   452→# application/dtos/user_dtos.py
   453→from dataclasses import dataclass
   454→from enum import Enum
   455→
   456→
   457→class UserStatusDTO(Enum):
   458→    ACTIVE = "active"
   459→    INACTIVE = "inactive"
   460→    BANNED = "banned"
   461→
   462→
   463→@dataclass
   464→class UserProfileDTO:
   465→    """Read-only projection of user data for presentation."""
   466→    user_id: int
   467→    telegram_id: int
   468→    username: str
   469→    role: str
   470→    status: UserStatusDTO
   471→    conversation_count: int
   472→    total_messages: int
   473→    registered_at: str
   474→
   475→
   476→@dataclass
   477→class UserListDTO:
   478→    """Paginated user list for admin views."""
   479→    users: list[UserProfileDTO]
   480→    total_count: int
   481→    page: int
   482→    per_page: int
   483→    has_next: bool
   484→```
   485→
   486→### Ports
   487→
   488→Ports are abstract interfaces defined by the application layer. They represent the operations the application needs from the outside world. There are two kinds:
   489→
   490→- **Driven ports** (secondary): Interfaces the application uses to interact with external systems (databases, LLMs, event buses)
   491→- **Driving ports** (primary): Interfaces through which external actors invoke the application (less common in Python, more explicit in Java/Spring)
   492→
   493→```python
   494→# application/ports/repositories.py
   495→from abc import ABC, abstractmethod
   496→from uuid import UUID
   497→
   498→from domain.entities.user import User
   499→from domain.entities.chat import Chat
   500→from domain.entities.conversation import Conversation
   501→
   502→
   503→class UserRepository(ABC):
   504→    """Driven port for user persistence."""
   505→
   506→    @abstractmethod
   507→    async def get_by_id(self, user_id: int) -> User | None: ...
   508→
   509→    @abstractmethod
   510→    async def get_by_telegram_id(self, telegram_id: int) -> User | None: ...
   511→
   512→    @abstractmethod
   513→    async def save(self, user: User) -> User: ...
   514→
   515→    @abstractmethod
   516→    async def get_all_active(self, offset: int, limit: int) -> list[User]: ...
   517→
   518→    @abstractmethod
   519→    async def count_active(self) -> int: ...
   520→
   521→
   522→class ChatRepository(ABC):
   523→    """Driven port for chat persistence."""
   524→
   525→    @abstractmethod
   526→    async def get_by_telegram_id(self, telegram_id: int) -> Chat | None: ...
   527→
   528→    @abstractmethod
   529→    async def save(self, chat: Chat) -> Chat: ...
   530→
   531→
   532→class ConversationRepository(ABC):
   533→    """Driven port for conversation persistence."""
   534→
   535→    @abstractmethod
   536→    async def get_by_id(self, conversation_id: UUID) -> Conversation | None: ...
   537→
   538→    @abstractmethod
   539→    async def get_active_by_user(self, telegram_id: int) -> Conversation | None: ...
   540→
   541→    @abstractmethod
   542→    async def save(self, conversation: Conversation) -> Conversation: ...
   543→
   544→    @abstractmethod
   545→    async def count_by_user(self, telegram_id: int) -> int: ...
   546→```

   549→
   550→## Infrastructure Layer
   551→
   552→The Infrastructure Layer provides concrete implementations of the ports defined by the application layer. It contains all code that interacts with external systems: databases, message queues, LLM APIs, Redis, and the Telegram Bot API itself.
   553→
   554→### Repository Implementations
   555→
   556→```python
   557→# adapters/database/user_repository.py
   558→from sqlalchemy.ext.asyncio import AsyncSession
   559→from sqlalchemy import select, func
   560→
   561→from application.ports.repositories import UserRepository
   562→from domain.entities.user import User, UserRole
   563→from adapters.database.models import UserModel
   564→
   565→
   566→class SQLAlchemyUserRepository(UserRepository):
   567→    """Concrete implementation of UserRepository using SQLAlchemy."""
   568→
   569→    def __init__(self, session: AsyncSession) -> None:
   570→        self._session = session
   571→
   572→    async def get_by_id(self, user_id: int) -> User | None:
   573→        result = await self._session.execute(
   574→            select(UserModel).where(UserModel.id == user_id)
   575→        )
   576→        row = result.scalar_one_or_none()
   577→        return self._to_entity(row) if row else None
   578→
   579→    async def get_by_telegram_id(self, telegram_id: int) -> User | None:
   580→        result = await self._session.execute(
   581→            select(UserModel).where(UserModel.telegram_id == telegram_id)
   582→        )
   583→        row = result.scalar_one_or_none()
   584→        return self._to_entity(row) if row else None
   585→
   586→    async def save(self, user: User) -> User:
   587→        if user.id == 0:
   588→            model = UserModel(
   589→                telegram_id=user.telegram_id,
   590→                username=user.username,
   591→                role=user.role.value,
   592→                is_active=user.is_active,
   593→            )
   594→            self._session.add(model)
   595→            await self._session.flush()
   596→            return self._to_entity(model)
   597→        else:
   598→            result = await self._session.execute(
   599→                select(UserModel).where(UserModel.id == user.id)
   600→            )
   601→            model = result.scalar_one()
   602→            model.username = user.username
   603→            model.role = user.role.value
   604→            model.is_active = user.is_active
   605→            await self._session.flush()
   606→            return self._to_entity(model)
   607→
   608→    async def get_all_active(self, offset: int, limit: int) -> list[User]:
   609→        result = await self._session.execute(
   610→            select(UserModel)
   611→            .where(UserModel.is_active == True)
   612→            .offset(offset)
   613→            .limit(limit)
   614→        )
   615→        return [self._to_entity(row) for row in result.scalars().all()]
   616→
   617→    async def count_active(self) -> int:
   618→        result = await self._session.execute(
   619→            select(func.count()).where(UserModel.is_active == True)
   620→        )
   621→        return result.scalar_one()
   622→
   623→    @staticmethod
   624→    def _to_entity(model: UserModel) -> User:
   625→        return User(
   626→            id=model.id,
   627→            telegram_id=model.telegram_id,
   628→            username=model.username,
   629→            role=UserRole(model.role),
   630→            is_active=model.is_active,
   631→            created_at=model.created_at,
   632→        )
   633→```

   635→### LLM Service Implementation
   636→
   637→```python
   638→# adapters/llm/openai_service.py
   639→from uuid import UUID
   640→import openai
   641→
   642→from application.use_cases.send_ai_message import LLMService
   643→
   644→
   645→class OpenAILLMService(LLMService):
   646→    """Concrete implementation of LLMService using OpenAI API."""
   647→
   648→    def __init__(self, api_key: str, model: str = "gpt-4o-mini") -> None:
   649→        self._client = openai.AsyncOpenAI(api_key=api_key)
   650→        self._model = model
   651→        self._conversation_history: dict[UUID, list[dict]] = {}
   652→
   653→    async def generate_response(
   654→        self,
   655→        conversation_id: UUID,
   656→        user_message: str,
   657→    ) -> str:
   658→        if conversation_id not in self._conversation_history:
   659→            self._conversation_history[conversation_id] = [
   660→                {"role": "system", "content": "You are a helpful Telegram bot assistant."}
   661→            ]
   662→
   663→        self._conversation_history[conversation_id].append(
   664→            {"role": "user", "content": user_message}
   665→        )
   666→
   667→        response = await self._client.chat.completions.create(
   668→            model=self._model,
   669→            messages=self._conversation_history[conversation_id],
   670→            max_tokens=2048,
   671→            temperature=0.7,
   672→        )
   673→
   674→        assistant_message = response.choices[0].message.content
   675→        self._conversation_history[conversation_id].append(
   676→            {"role": "assistant", "content": assistant_message}
   677→        )
   678→
   679→        return assistant_message
   680→```

   683→
   684→```python
   685→# adapters/events/redis_event_publisher.py
   686→from domain.events.domain_events import DomainEvent
   687→from application.use_cases.start_conversation import EventPublisher
   688→import json
   689→from redis.asyncio import Redis
   690→
   691→
   692→class RedisEventPublisher(EventPublisher):
   693→    """Publishes domain events to Redis pub/sub channels."""
   694→
   695→    def __init__(self, redis: Redis) -> None:
   696→        self._redis = redis
   697→
   698→    async def publish(self, event: DomainEvent) -> None:
   699→        channel = f"events:{type(event).__name__}"
   700→        payload = {
   701→            "event_type": type(event).__name__,
   702→            "occurred_at": event.occurred_at.isoformat(),

   704→                k: str(v) for k, v in event.__dict__.items()
   705→                if k != "occurred_at"
   706→            },
   707→        }
   708→        await self._redis.publish(channel, json.dumps(payload))

   713→## Presentation Layer
   714→
   715→The Presentation Layer contains the controllers (aiogram handlers), presenters (response formatters), keyboards, and filters. This layer is the bridge between the Telegram Bot API and your application's use cases.
   716→
   717→### Controllers (Message/Callback Handlers)
   718→
   719→Controllers are thin — they parse incoming Telegram updates, call use cases, and delegate response formatting to presenters.
   720→
   721→```python
   722→# adapters/telegram/controllers/chat_controller.py
   723→from aiogram import Router
   724→from aiogram.types import Message, CallbackQuery
   725→from aiogram.filters import Command
   726→
   727→from application.use_cases.start_conversation import (
   728→    StartConversationUseCase,
   729→    StartConversationRequest,
   730→)
   731→from application.use_cases.send_ai_message import (
   732→    SendAIMessageUseCase,
   733→    SendAIMessageRequest,
   734→)
   735→from adapters.telegram.presenters.chat_presenter import ChatPresenter
   736→
   737→
   738→router = Router()
   739→
   740→
   741→@router.message(Command("chat"))
   742→async def start_conversation(
   743→    message: Message,
   744→    start_conversation_use_case: StartConversationUseCase,
   745→) -> None:
   746→    """Controller: starts a new AI conversation."""
   747→    request = StartConversationRequest(
   748→        user_telegram_id=message.from_user.id,
   749→    )
   750→    response = await start_conversation_use_case.execute(request)
   751→
   752→    presenter = ChatPresenter()
   753→    text = presenter.format_conversation_start(response)
   754→    keyboard = presenter.build_conversation_keyboard(response)
   755→
   756→    await message.answer(text, reply_markup=keyboard)
   757→
   758→
   759→@router.message()
   760→async def handle_chat_message(
   761→    message: Message,
   762→    send_ai_message_use_case: SendAIMessageUseCase,
   763→    active_conversation_id: str | None,
   764→) -> None:
   765→    """Controller: processes a user message in an active AI conversation."""
   766→    if not active_conversation_id:
   767→        await message.answer(
   768→            "No active conversation. Use /chat to start one."
   769→        )
   770→        return
   771→
   772→    request = SendAIMessageRequest(
   773→        conversation_id=active_conversation_id,
   774→        user_telegram_id=message.from_user.id,
   775→        message_text=message.text,
   776→    )
   777→
   778→    response = await send_ai_message_use_case.execute(request)
   779→
   780→    presenter = ChatPresenter()
   781→    text = presenter.format_ai_response(response)
   782→    await message.answer(text)
   783→```
   784→
   785→### Presenters (Response Formatters)
   786→
   787→Presenters transform use case responses into human-readable Telegram messages. They handle markdown formatting, emoji, and keyboard layout.
   788→
   789→```python
   790→# adapters/telegram/presenters/chat_presenter.py
   791→from aiogram.types import InlineKeyboardMarkup, InlineKeyboardButton
   792→
   793→from application.use_cases.start_conversation import StartConversationResponse
   794→from application.use_cases.send_ai_message import SendAIMessageResponse
   795→
   796→
   797→class ChatPresenter:
   798→    """Formats AI chat responses for Telegram display."""
   799→
   800→    def format_conversation_start(self, response: StartConversationResponse) -> str:
   801→        if response.is_resumed:
   802→            return (
   803→                "💬 *Conversation Resumed*\n\n"
   804→                f"Remaining messages: {response.remaining_messages}\n"
   805→                "Type your message to continue."
   806→            )
   807→        return (
   808→            "💬 *New Conversation Started*\n\n"
   809→            "I'm ready to chat! Ask me anything.\n"
   810→            f"You have {response.remaining_messages} messages available.\n"
   811→            "Use /end to close this conversation."
   812→        )
   813→
   814→    def build_conversation_keyboard(
   815→        self, response: StartConversationResponse
   816→    ) -> InlineKeyboardMarkup:
   817→        buttons = [
   818→            [
   819→                InlineKeyboardButton(
   820→                    text="🔚 End Conversation",
   821→                    callback_data=f"end_conversation:{response.conversation_id}",
   822→                )
   823→            ],
   824→        ]
   825→        if response.remaining_messages > 10:
   826→            buttons.append([
   827→                InlineKeyboardButton(
   828→                    text="⚙️ Settings",
   829→                    callback_data="conversation_settings",
   830→                )
   831→            ])
   832→        return InlineKeyboardMarkup(inline_keyboard=buttons)
   833→
   834→    def format_ai_response(self, response: SendAIMessageResponse) -> str:
   835→        footer = f"\n\n_Messages remaining: {response.remaining_messages}_"
   836→        if len(response.ai_response) + len(footer) > 4096:
   837→            return response.ai_response[:4090 - len(footer)] + "..." + footer
   838→        return response.ai_response + footer
   839→```

   900→## Cross-Cutting Concerns
   901→
   902→Cross-cutting concerns are aspects of the application that span multiple layers. In a Clean Architecture Telegram bot, these include logging, error handling, dependency injection, and configuration.
   903→
   904→### Logging
   905→
   906→Logging must be structured and consistent. Define a logging protocol in the application layer and implement it in infrastructure.
   907→
   908→```python
   909→# application/ports/logger.py
   910→from abc import ABC, abstractmethod
   911→from typing import Any
   912→
   913→
   914→class AppLogger(ABC):
   915→    """Application-level logging port."""
   916→
   917→    @abstractmethod
   918→    def info(self, message: str, **kwargs: Any) -> None: ...
   919→
   920→    @abstractmethod
   921→    def error(self, message: str, **kwargs: Any) -> None: ...
   922→
   923→    @abstractmethod
   924→    def warning(self, message: str, **kwargs: Any) -> None: ...
   925→
   926→    @abstractmethod
   927→    def debug(self, message: str, **kwargs: Any) -> None: ...
   928→
   929→
   930→# infrastructure/logging/logger.py
   931→import logging
   932→import structlog
   933→
   934→
   935→class StructlogLogger(AppLogger):
   936→    """Concrete logger using structlog for structured JSON output."""
   937→
   938→    def __init__(self, service_name: str) -> None:
   939→        structlog.configure(
   940→            processors=[
   941→                structlog.contextvars.merge_contextvars,
   942→                structlog.processors.add_log_level,
   943→                structlog.processors.TimeStamper(fmt="iso"),
   944→                structlog.processors.JSONRenderer(),
   945→            ],
   946→        )
   947→        self._logger = structlog.get_logger(service=service_name)
   948→
   949→    def info(self, message: str, **kwargs: Any) -> None:
   950→        self._logger.info(message, **kwargs)
   951→
   952→    def error(self, message: str, **kwargs: Any) -> None:
   953→        self._logger.error(message, **kwargs)
   954→
   955→    def warning(self, message: str, **kwargs: Any) -> None:
   956→        self._logger.warning(message, **kwargs)
   957→
   958→    def debug(self, message: str, **kwargs: Any) -> None:
   959→        self._logger.debug(message, **kwargs)
   960→```

   962→### Error Handling
   963→
   964→Define a consistent error handling strategy that translates domain exceptions into user-friendly Telegram messages.
   965→
   966→```python
   967→# adapters/telegram/error_handler.py
   968→from aiogram.types import Update
   969→from aiogram import Router
   970→
   971→from domain.exceptions.domain_exceptions import (
   972→    ChatAlreadyClosedError,
   973→    ConversationAlreadyEndedError,
   974→    MessageLimitExceededError,
   975→)
   976→
   977→
   978→router = Router()
   979→
   980→
   981→@router.error()
   982→async def handle_errors(update: Update, exception: Exception) -> None:
   983→    """Global error handler for the bot."""
   984→
   985→    error_messages = {
   986→        ChatAlreadyClosedError: "This chat is no longer active.",
   987→        ConversationAlreadyEndedError: "This conversation has ended. Start a new one with /chat.",
   988→        MessageLimitExceededError: "You've reached the message limit for this conversation. Start a new one with /chat.",
   989→        PermissionError: "You don't have permission to perform this action.",
   990→        ValueError: "Invalid input. Please check your request and try again.",
   991→    }
   992→
   993→    message_text = error_messages.get(type(exception), "An unexpected error occurred. Please try again later.")
   994→
   995→    if update.message:
   996→        await update.message.answer(f"⚠️ {message_text}")
   997→    elif update.callback_query:
   998→        await update.callback_query.answer(f"⚠️ {message_text}", show_alert=True)
   999→```

  1002→
  1003→The composition root wires all dependencies together. This is the single place where concrete implementations are bound to abstract interfaces.
  1004→

  1006→# infrastructure/container.py
  1007→"""Composition root: wires all dependencies together."""
  1008→

  1010→from infrastructure.database.session import create_session_factory

  1012→from adapters.database.chat_repository import SQLAlchemyChatRepository

  1063→            user_repo=self.user_repository(),
  1064→            referral_service=...,  # wired similarly
  1065→        )
  1066→
  1067→    def start_conversation_use_case(self) -> StartConversationUseCase:

  1072→

  1120→## Data Flow Across Layers
  1121→
  1122→Understanding how data transforms as it crosses layer boundaries is critical. Here is the complete flow for a user sending a message to the AI:
  1123→
  1124→```
  1125→1. Telegram API → aiogram receives Update
  1126→2. Framework Layer (aiogram Router) dispatches to handler
  1127→3. Presentation Layer:
  1128→   - Controller parses Message → creates SendAIMessageRequest DTO
  1129→   - Calls SendAIMessageUseCase.execute(request)
  1130→4. Application Layer:
  1131→   - Use case loads Conversation via ConversationRepository port
  1132→   - Validates message via MessageText value object
  1133→   - Calls LLMService.generate_response() port
  1134→   - Saves updated Conversation via repository port
  1135→   - Publishes MessageSentToLLM event via EventPublisher port
  1136→   - Returns SendAIMessageResponse DTO
  1137→5. Presentation Layer:
  1138→   - Presenter formats response DTO into Telegram markdown
  1139→   - Handler calls message.answer(formatted_text)
  1140→6. Framework Layer:
  1141→   - aiogram sends API request to Telegram servers
  1142→   - User receives the AI response
  1143→```
  1144→
  1145→At each boundary crossing, data is transformed from one representation to another:
  1146→- **Telegram Update → Request DTO**: Controller extracts relevant fields
  1147→- **Request DTO → Domain Entities**: Use case reconstructs domain objects from repository
  1148→- **Domain Entities → Response DTO**: Use case maps entity state to output structure
  1149→- **Response DTO → Telegram Message**: Presenter formats for human consumption
  1150→
  1151→This transformation at each boundary is what makes Clean Architecture powerful — each layer communicates in its own language, optimized for its own concerns.
  1152→

  1155→## Common Pitfalls by Layer
  1156→
  1157→### Domain Layer Pitfalls
  1158→
  1159→- **Anemic entities**: Entities with only getters and setters, no business logic. Push validation and state transitions into entities.
  1160→- **Framework leakage**: Importing SQLAlchemy, Pydantic, or aiogram types in entity classes. Entities must be pure Python.
  1161→- **Missing value objects**: Using raw `int` for telegram IDs or raw `str` for messages instead of dedicated value objects with validation.
  1162→
  1163→### Application Layer Pitfalls
  1164→
  1165→- **Use cases with infrastructure logic**: Directly importing database or LLM clients instead of using port interfaces.
  1166→- **Fat DTOs**: Putting validation or business logic in DTOs. DTOs should be dumb data carriers.
  1167→- **Missing error handling**: Not defining what happens when a port raises an exception. Use cases should handle expected failures gracefully.
  1168→
  1169→### Infrastructure Layer Pitfalls
  1170→
  1171→- **Leaking ORM models**: Returning SQLAlchemy model objects instead of domain entities from repositories. Always convert in the adapter.
  1172→- **Tight coupling to specific vendor**: Hard-coding OpenAI-specific parameters in the LLM service. Keep vendor specifics behind the adapter.
  1173→
  1174→### Presentation Layer Pitfalls
  1175→
  1176→- **Business logic in handlers**: Handlers should parse input and format output only. Any decision-making belongs in use cases.
  1177→- **Skipping the presenter**: Formatting messages directly in handlers couples presentation to control flow.
  1178→- **Over-complex keyboard builders**: Keyboard logic should be simple and delegatable to a dedicated module.
  1179→
